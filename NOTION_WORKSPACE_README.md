## Workflow Recommendations

### Daily Workflow
1. Check "My Tasks" view
2. Move one task to "In Progress"
3. Work on that task exclusively
4. Mark complete when done
5. Update task with notes
6. Move next task to "In Progress"

### Weekly Workflow
1. Review "Sprint Planning" view
2. Prioritize P0 and P1 tasks
3. Assign tasks to team members
4. Set due dates for the week

1. Use 2-week sprints
2. Group related tasks (e.g., all Property tasks)
3. Set sprint goals aligned with milestones
4. Track velocity (tasks completed per sprint)

## Troubleshooting

### Common Issues

**Issue**: CSV import fails
**Solution**: Create the database first, then use "Merge with CSV"

**Issue**: Properties show as text
**Solution**: Manually change property types after import

**Issue**: Board view doesn't work
**Solution**: Ensure Status is a Select property type

**Issue**: Formulas show errors
**Solution**: Check property names match exactly (case-sensitive)

**Issue**: Tasks aren't sorting correctly
**Solution**: Add Priority Score formula and sort by it

## Success Metrics

### Process Metrics
- **Velocity**: Tasks completed per week/sprint
- **Cycle Time**: Average time from "To Do" to "Completed"
- **Lead Time**: Average time from creation to completion
- **Blocker Frequency**: How often tasks get blocked

### Project Metrics
- **Completion %**: (Completed / Total) √ó 100
- **On-Time Delivery**: % of tasks completed by due date
- **Scope Creep**: New tasks added vs. original plan
- **Priority Distribution**: Are we focusing on P0/P1?

### Quality Metrics
- **Bugs Found**: Track bugs discovered post-completion
- **Rework Rate**: % of tasks marked complete that need more work
- **Review Cycles**: Average number of review rounds per task
